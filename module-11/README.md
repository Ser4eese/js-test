## 📚 Теория

### Что такое Git?

**Git** — это система контроля версий. Представьте, что вы пишете курсовую работу и сохраняете файлы как `курсовая_v1.docx`, `курсовая_v2.docx`, `курсовая_финал.docx`, `курсовая_финал_точно.docx`. Git делает это автоматически и намного лучше.

Git запоминает:
- **Что** изменилось (какие строки добавлены/удалены)
- **Когда** изменилось (дата и время)
- **Кто** изменил (автор)
- **Зачем** изменил (сообщение коммита)

### Зачем нужен Git?

1. **История изменений** — можно вернуться к любой версии кода
2. **Командная работа** — несколько человек работают над одним проектом
3. **Эксперименты** — можно пробовать новое без страха сломать код
4. **Бэкап** — код хранится не только на вашем компьютере

### Основные понятия

```
┌─────────────────────────────────────────────────────────────┐
│                    Рабочая директория                        │
│                   (ваши файлы на диске)                      │
└─────────────────────────────────────────────────────────────┘
                           │
                           │ git add
                           ▼
┌─────────────────────────────────────────────────────────────┐
│                    Staging Area (Индекс)                     │
│               (файлы, готовые к коммиту)                     │
└─────────────────────────────────────────────────────────────┘
                           │
                           │ git commit
                           ▼
┌─────────────────────────────────────────────────────────────┐
│                       Репозиторий                            │
│                  (история всех коммитов)                     │
└─────────────────────────────────────────────────────────────┘
```

- **Репозиторий (repo)** — папка проекта, которую отслеживает Git
- **Коммит (commit)** — снимок состояния файлов в определённый момент
- **Staging area** — область подготовки файлов к коммиту

### Основные команды

| Команда | Описание |
|---------|----------|
| `git init` | Создать новый репозиторий |
| `git status` | Показать состояние файлов |
| `git add <файл>` | Добавить файл в staging area |
| `git add .` | Добавить все файлы |
| `git commit -m "сообщение"` | Создать коммит |
| `git log` | Показать историю коммитов |
| `git log --oneline` | Короткая история |

### Пример рабочего процесса

```bash
# 1. Создаём репозиторий
git init

# 2. Создаём/редактируем файлы
echo "console.log('Hello');" > index.js

# 3. Проверяем статус
git status
# Untracked files: index.js

# 4. Добавляем в staging
git add index.js

# 5. Создаём коммит
git commit -m "Добавил index.js"

# 6. Смотрим историю
git log --oneline
# a1b2c3d Добавил index.js
```

## ❓ Частые вопросы

**Q: Зачем нужен `git add`, почему нельзя сразу `commit`?**

A: Staging area позволяет выбрать, какие изменения войдут в коммит. Изменили 5 файлов, но хотите закоммитить только 2 — добавляете только их.

**Q: Что такое `.git` папка?**

A: Скрытая папка, где Git хранит всю историю. Удалите её — и репозиторий перестанет быть репозиторием.

**Q: Чем Git отличается от GitHub?**

A: Git — это программа на вашем компьютере. GitHub — это сайт для хранения репозиториев онлайн и совместной работы.
